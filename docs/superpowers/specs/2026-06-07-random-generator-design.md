# 随机数生成器微信小程序 — 设计规格

## 概述

一个微信小程序，用于批量生成随机数并提供大小分类统计。核心痛点：原有工具生成大量随机数后，用户需要手动数数来验证大小统计，效率低。本小程序通过颜色标记 + 角标计数解决这个问题。

## 目标平台

微信小程序原生框架（WXML / WXSS / JS）。

## 设计风格

基于 silent-night-ui（Nocturne）暗色视觉系统，适配微信小程序：

- 深色背景（`#0a0d12`）、暗层卡片（`#14181e`）、大圆角
- 主 accent 色：mint 薄荷绿（`#45e6a4`），用于品牌标识和操作按钮
- **例外**：结果数字使用红/绿双色标记大小，突破单 accent 限制，功能优先
- 系统字体栈，无网络字体依赖
- 全中文界面

## 页面结构（单页，配置优先）

从上到下三个区域：

### 1. 顶栏

- 左侧：mint 绿点 + 「随机数生成器」品牌文字（11px，大写字母间距）
- 右侧：「历史记录」pill 按钮

### 2. 配置卡片

暗层卡片（`#14181e`，圆角 20rpx），包含：

- **section-label**：mint 圆点 + 「参数设置」标签
- **参数区**（2×2 网格，子面板 `#1a1f27`）：
  - 最小值：数字输入框，默认 0
  - 最大值：数字输入框，默认 100
  - 生成个数：数字输入框，默认 1
  - 允许重复：开关（Switch），默认开启，开启时 mint 色
- **操作按钮**（底部一行）：
  - 「重置」：次级按钮（透明底 + 细边框），flex:1
  - 「生成随机数」：主按钮（mint 实心），flex:2

### 3. 结果卡片（生成后展示）

暗层卡片，包含：

- **标题行**：section-label「生成结果」+ 右侧「复制结果」pill 按钮
- **数字网格**：
  - 每个数字一个方块（约 80rpx × 80rpx，圆角 10rpx）
  - 大数（≥ 分界点）：绿色背景（`rgba(76,175,80,0.15)`）+ 绿色文字（`#81c784`）
  - 小数（< 分界点）：红色背景（`rgba(239,83,80,0.15)`）+ 红色文字（`#ef9a9a`）
  - **角标**：每个方块右上角显示该类别的序号（第几个大数 / 第几个小数），颜色与数字同色但降低透明度（0.7），字号 8px
  - 网格 flex 布局，自动换行，间距 8rpx
- **统计区**（分割线下方）：
  - 标签：「大小统计（分界点：N）」
  - 两列统计面板：
    - 大（≥N）：数量（绿色大字 28px）+ 百分比
    - 小（<N）：数量（红色大字 28px）+ 百分比
  - 比例条：6px 高度的水平条，绿/红按比例填充

### 4. 空状态

未生成时，结果卡片显示居中灰色提示文字「请生成随机数」。

## 分界点规则

自动计算：`分界点 = Math.ceil((最小值 + 最大值) / 2)`

- 大于等于分界点 → 大（绿色）
- 小于分界点 → 小（红色）

## 历史记录弹窗

- **触发**：点击顶栏「历史记录」按钮
- **展示**：模态弹窗（半透明黑色遮罩 + 居中暗色面板）
- **每条记录**：
  - 参数摘要：「0 ~ 60 × 100个」
  - 时间戳：「22:15:03」格式（当天）或「06-07 22:15」格式（跨天）
  - 统计概览：大 N | 小 N | 允许重复/不重复
- **交互**：点击记录可恢复查看完整结果（将结果加载到结果卡片中）
- **存储**：使用微信小程序 `wx.setStorageSync` / `wx.getStorageSync` 本地存储
- **容量**：最多保留最近 20 条记录，超出时自动删除最旧的

## 复制结果

点击「复制结果」按钮，将所有生成的数字以逗号分隔的格式复制到剪贴板（使用 `wx.setClipboardData`），并显示微信原生的复制成功提示。

## 边界校验

- 最小值必须 ≤ 最大值，否则生成按钮禁用并提示
- 生成个数必须 ≥ 1
- 当「允许重复」关闭时，生成个数不能超过范围内整数个数（最大值 - 最小值 + 1），超出时提示用户调整
- 所有输入框只接受非负整数

## 单位说明

WXSS 中统一使用 rpx（响应式像素）。本文档中标注的 px 值为视觉参考，实现时按 1px ≈ 2rpx 换算。

## 不包含的功能

- 配置保存/预设
- 分享结果
- 奇偶统计、排序显示等额外分析
- 多主题切换（dark-only）
- 网络请求 / 后端服务
